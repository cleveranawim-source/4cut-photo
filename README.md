# 우리들의 네컷사진관

iPad로 네 장을 연속 촬영하고, 4×6 인화용 네컷 사진을 만든 뒤 QR 또는 AirPrint로 나누는 학교 행사·학급 활동용 웹앱입니다.

## 현재 들어 있는 기능

- 전면 카메라 미리보기와 3초 카운트다운
- 포즈를 바꿔가며 네 장 자동 촬영
- 살구 크림·베리 팝·미드나잇 프레임
- 1200×1800픽셀(4×6인치, 300dpi 기준) 인화 이미지 생성
- 한 장에 같은 2×6 스트립 두 개 배치 — 인화 후 가운데를 잘라 두 장으로 사용
- QR 다운로드 링크 생성
- QR 사진 기본 60분 후 자동 삭제
- AirPrint용 4×6 인쇄 레이아웃
- iPad 저장·공유 및 다시 찍기
- 홈 화면에 추가할 수 있는 PWA

## 컴퓨터에서 바로 실행

Node.js 20 이상이 필요합니다.

```powershell
npm install
npm run build
npm start
```

브라우저에서 `http://localhost:4173`을 엽니다. 카메라 없이 전체 흐름을 확인하려면 첫 화면의 **카메라 없이 샘플로 둘러보기**를 누르세요.

개발 중에는 아래 명령으로 화면과 서버를 동시에 실행할 수 있습니다.

```powershell
npm run dev
```

- 화면: `http://localhost:4173`
- QR API: `http://localhost:4174`

## iPad에서 사용하려면

iPadOS는 웹 카메라를 HTTPS 주소에서만 허용합니다. 이 프로젝트를 HTTPS가 적용된 Node.js 호스팅에 배포한 다음 아래 순서로 설치합니다.

1. iPad의 Safari에서 배포 주소를 엽니다.
2. 공유 버튼을 누르고 **홈 화면에 추가**를 선택합니다.
3. 생성된 **네컷사진관** 아이콘을 실행합니다.
4. 처음 한 번 카메라 사용을 허용합니다.
5. QR 기능을 쓸 때는 iPad가 인터넷에 연결되어 있어야 합니다.

`.env.example`을 참고해 운영 환경 변수를 설정하세요. 외부 주소를 자동으로 정확히 인식하지 못하는 호스팅에서는 `PUBLIC_BASE_URL`을 실제 HTTPS 주소로 지정해야 QR이 올바른 주소를 만듭니다.

## Railway 배포

프로젝트에는 `railway.json`이 포함되어 있어 Railway가 자동으로 빌드·실행하고 `/api/health`로 상태를 확인합니다.

```powershell
railway login --browserless
railway init
railway up
railway domain
railway volume add --mount-path /data
railway variable set STORAGE_DIR=/data SHARE_TTL_MINUTES=60
```

볼륨은 QR 사진이 배포·재시작 사이에도 유지되도록 합니다. 무료 플랜의 기본 볼륨 한도는 0.5GB이며, 이 앱은 60분이 지난 사진을 자동 정리하므로 시험 운영에 충분합니다. 공개 주소가 만들어진 뒤 `PUBLIC_BASE_URL=https://생성주소.up.railway.app`도 설정하고 한 번 재배포하세요.

## AirPrint 프린터 연결

1. iPad와 AirPrint 지원 포토프린터를 같은 Wi‑Fi에 연결합니다.
2. 완성 화면에서 **사진 인쇄하기**를 누릅니다.
3. AirPrint 창에서 프린터와 4×6(Postcard) 용지를 선택합니다.
4. 가능하면 여백 없음, 컬러, 세로 방향으로 인쇄합니다.

학교 Wi‑Fi가 기기 간 통신을 차단하면 프린터가 검색되지 않을 수 있습니다. 행사 전용 공유기 한 대에 iPad와 프린터를 함께 연결하고, 공유기의 인터넷 연결도 확보하는 구성이 가장 안정적입니다. Canon SELPHY CP1500처럼 AirPrint와 4×6 엽서 용지를 모두 지원하는 승화형 포토프린터가 이 레이아웃에 잘 맞습니다.

## QR 사진 보관 방식

- QR 버튼을 누르기 전까지 촬영 결과는 브라우저 메모리에만 있습니다.
- QR 버튼을 누르면 48자리 무작위 토큰의 JPEG 파일이 서버 `storage/`에 저장됩니다.
- 파일 이름이나 URL에 학생 이름을 넣지 않습니다.
- 기본 60분이 지나면 요청 시 즉시 삭제되고, 서버 정리 작업도 주기적으로 삭제합니다.
- 검색 엔진 색인을 막고, QR 페이지와 이미지 응답에 캐시 금지 헤더를 적용합니다.

실제 학교 운영 전에는 학교의 촬영·개인정보 처리 지침과 보호자 동의 범위를 확인하세요. 서버 관리자에게는 파일 접근 권한이 있으므로, 공개 행사나 장기 운영에서는 저장 암호화와 접근 로그 정책을 추가하는 것이 좋습니다.

## 운영 시 알아둘 점

- 이 1차 버전은 웹앱이므로 인쇄할 때마다 iPad의 표준 AirPrint 창이 열립니다. 프린터를 기억하고 완전한 원터치 인쇄를 하려면 후속 네이티브 iPad 앱이 필요합니다.
- 사진은 현재 서버 한 대의 로컬 디스크에 저장됩니다. 여러 서버로 자동 확장하는 환경에서는 공유 스토리지(S3/R2 등)로 교체해야 합니다.
- 서버가 재시작되어도 파일은 남아 있으면 기존 만료 시간이 적용됩니다. 임시 디스크를 쓰는 호스팅은 재시작 시 사진이 더 일찍 사라질 수 있습니다.
- 프린터 자체 Wi‑Fi Direct에만 연결하면 iPad가 인터넷을 잃어 QR 업로드가 실패할 수 있습니다. 인터넷이 연결된 별도 공유기를 권장합니다.

## 프로젝트 구조

```text
src/App.tsx        촬영, 합성, QR, 인쇄 화면
src/styles.css     iPad 반응형 화면과 4×6 인쇄 CSS
server.mjs         QR 업로드, 다운로드, 자동 만료 서버
public/            PWA 설치 파일과 아이콘
storage/           실행 중 생기는 임시 사진(버전 관리 제외)
```
